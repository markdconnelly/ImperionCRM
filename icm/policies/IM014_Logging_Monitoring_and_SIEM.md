## 1. Purpose

To define requirements for the generation, collection, retention, monitoring, and analysis of security-relevant logs so that threats are detected and responded to promptly across Imperion and client environments.

## 2. Definitions

- **SIEM** — Security Information and Event Management. Imperion uses **Microsoft Sentinel**, integrated with **Microsoft Defender XDR**.
- **XDR** — Extended Detection and Response across endpoint, identity, email, and cloud apps (Microsoft Defender XDR).
- **SOAR** — Security Orchestration, Automation, and Response capabilities used to automate detection handling.
- **Telemetry** — Security-relevant event data generated by systems and services.

## 3. Statement of Policy

### 3.1 Centralized Logging Architecture

- Security telemetry is centralized in **Microsoft Sentinel**, which ingests **DNS, Microsoft Entra, and Azure activity logs**, Defender XDR signals, and other relevant sources.
- Sentinel is **integrated with Defender XDR** to provide unified detection, investigation, and response across endpoint, identity, email, and cloud.
- Where technically possible, security logs from mission-critical and client systems are forwarded to Sentinel.

### 3.2 Required Log Sources and Events

At minimum, the following are logged and retained:

- Authentication events (success/failure), MFA challenges, and identity-risk events (Entra ID).
- Privileged-account use and administrative actions, including delegated (GDAP) client-tenant actions.
- Access to Restricted/Confidential data where feasible (IM012).
- Endpoint detections and response actions (Defender for Endpoint).
- Email threats and DLP events (Defender for Office 365, Purview).
- Network/DNS activity and firewall events.
- Cloud configuration changes (Azure activity logs).
- AI system activity and agent actions (IM011).
- System start/stop, and approved emergency access.

### 3.3 Time Synchronization and Integrity

- System clocks are synchronized to an authoritative time source so events correlate accurately.
- Logs are protected against tampering and unauthorized access; access to logging systems is least-privilege.

### 3.4 Retention

- Security logs are retained for a minimum of **one year**, with at least 90 days immediately searchable, unless a longer period is required by contract, law, or regulation.
- Client log retention follows the applicable client agreement.

### 3.5 Monitoring, Detection, and Alerting

- Sentinel analytics rules and Defender XDR detections generate alerts that are triaged on a defined schedule.
- High-severity detections trigger the Technical Incident Response Program.
- Detection content is tuned continuously to reduce false positives and address emerging threats.
- Automation/SOAR playbooks are used to accelerate containment where appropriate.

### 3.6 Continuous Monitoring Coverage

Monitoring includes, but is not limited to: perimeter and firewall activity, intrusion detection alerts, user and identity activity, identity-risk events, account-termination timeliness, application/database account activity, building-access anomalies, outbound DLP, inbound spam thresholds, AI system activity, and vulnerability-scan results. Exceptions or violations detected are recorded as security incidents with a documented risk assessment (IM006).

### 3.7 Reporting

Monitoring results and security posture are reported to the CISO at least annually and summarized for clients in quarterly business reviews (QBRs).
